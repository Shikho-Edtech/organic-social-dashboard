Cycle 1.5 · additions to Cycle 1 · 2026-04-21

Improvements on Cycle 1 + Login + a global page template

Three parts: (A) things I'd tighten on the Cycle 1 banner/empty-state before you port it, (B) a full redesign of the login page consistent with the Shikho brand and the banner vocabulary, (C) a unified page template + global patterns so every future page inherits the Cycle 1 decisions for free.

A · refinements on Cycle 1

Seven things I'd change before porting

These are all small. None invalidate Cycle 1 — they're the polish layer that comes from reading it back a day later.

A1 · stage registry

Pull stage metadata into one file

Banner, empty state, and detail panel all hardcode the stage name + env-var names. One lib/stages.ts with { id, label, noun, envVars, artifact } means the pattern scales to a 6th stage without copy-paste drift.

export const STAGES = {
  diagnosis: { label: "AI diagnosis",
    envVars: ["DIAGNOSIS_PROVIDER", ...],
    pages: ["/strategy"] },
  calendar:  { label: "AI calendar", ... },
} as const;
A2 · collapsed banner · crit

Ship a "what to do" affordance inline

At crit, the collapsed banner currently just says "tap for detail". One extra inline action — "Re-run pipeline" (link to GH Actions) — would convert the banner from a status indicator into a fix surface. Same pattern at warn.

A3 · detail panel · age sorting

Sort stages by severity, not pipeline order

On desktop the table currently mirrors pipeline execution order. When two stages are stale, the reader has to scan for the amber chips. Sort crit → warn → ok (stable within each bucket) so failures float to the top. Mobile stack already has this by virtue of being short.

A4 · empty state · env-var UX

Copy-to-clipboard on each env-var chip

The three env-var names are the actual action. Render each as a clickable chip that copies on tap with a 1.5s "Copied" toast inline. Saves a round-trip to the repo when setting secrets.

A5 · archival mode

Dim archived content 4%

Archival text line is right — but the content below looks identical to a live render. Drop the whole archived region to opacity-96 + filter saturate-75. A reader glancing back at the tab after a context-switch gets a peripheral cue the view is frozen, without reading copy.

A6 · animation

180ms ease-out on expand, nothing else

Current design has zero animation. One concession: transition: grid-template-rows 180ms on the detail panel (via the grid-rows 0fr/1fr trick). Makes the expand feel intentional instead of jumpy. Chevron rotation already in Cycle 1.

A7 · edge cases Cycle 1 didn't cover

Three states you'll hit in production

  • Never run. First deploy, stage has zero history. Empty state applies, but the copy "last successful run: Apr 11" needs a null-path: "No runs yet — the next weekly pipeline will populate this."
  • Partial success. Diagnose succeeded but Calendar errored on the same run. Banner should lead with Calendar (worse stage), and the detail panel's header should say "1 of 5 stages stale" not "pipeline healthy".
  • Graceful-degradation fallback. Your pipeline already has this (Day 2M/2N per CLAUDE.md). When a stage used fallback but succeeded, chip should say "ok · fallback" in amber, not solid ok — the view isn't stale but the reader should know Claude didn't run this week.

C · global template

One template every page inherits

The design audit flagged "no consistent page template" as a large-effort item. Cycle 1 gives you the banner/empty vocabulary — this part defines the shape every page above hangs on, so Trends / Engagement / Timing / Reels / Explore all feel like one product.

C1 · the five-row template
1StalenessBannerConsolidated, Cycle 1. Always above PageHeader. Absent on ok silently.
2PageHeaderTitle + subtitle left, DateRangePicker + dateLabel right. Never deviates.
3KPI strip2-up mobile, 4-6 up desktop. Each is a KpiCard. Max 6.
4Primary chart(s)Single full-width, or 2-up grid. Always ChartCard. Biggest visual anchor.
5Detail / breakdownTables, horizontal bars, or progressive-disclosure lists. Lower visual weight.

Rule: if a section on a new page doesn't fit one of these five slots, it belongs on a different page. This is how you shrink 8 pages to 5 without losing content — force the question at authoring time.

C2 · global patterns that apply everywhere

Color

Indigo-first, not cyan-first

Default bar + line color → #1e2a78. Cyan drops to secondary. Lets every chart lead with brand without override.

Loading

Skeleton per template slot

One loading.tsx at app root: banner placeholder + header skeleton + 6 KPI blocks + 2 chart blocks. Every page gets it free via Next.js.

Contrast

Kill every text-slate-400

Global find/replace → text-slate-500 for content. text-slate-600 for body paragraph weights. AA compliance in one commit.

Empty

One <EmptyChart> component

Currently ad-hoc divs in 5+ places. One component with icon + message + optional action — used by ChartCard automatically when data is empty.

Density

16 px gap between cards, universally

Grid gap-4 on every page. !p-5 on KPIs, p-6 on ChartCards. Never deviate — inconsistency is 80% of why dashboards feel "off".

Nav

Reduce 8 tabs to 5

Overview · Content · Timing · Strategy · Explore. Merge Trends into Overview, Reels into Content, Plan into Strategy as a "Next week" section. Mobile dropdown becomes readable.

Focus

Already shipped

Your globals.css focus-visible rule is right. Extend to the nav tabs (currently just border-bottom, no ring) and the DateRangePicker trigger.

Data line

Rename "Rendered" → "Data as of"

Render time is irrelevant to the reader. Use the last-successful-pipeline time from Analysis_Log. Matches the banner's vocabulary.

Motion

150-180ms ease-out, nowhere else

Banner expand (A6), mobile nav dropdown, DateRangePicker popup, tooltip fade. No chart animation, no page transitions, no scroll effects.
