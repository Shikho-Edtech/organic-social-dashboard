Internal design review · 18 Apr 2026

A trustworthy dashboard that doesn't yet feel like a product.

The organic social dashboard reads clearly, respects mobile, and treats data provenance as a first-class concern — which is rare. What's missing is the polish layer that would turn "competent internal tool" into something the team shows off. This assessment covers visual system, information architecture, accessibility, and chart choices, with before/after mockups for the four pages that carry the most weight.

02 — Top 5 strengths & opportunities

What's working, and what's holding it back.

Keep the first column. The second column is this assessment's playbook.

✓ Strengths

  1. Data provenance is a first-class design element. The kind prop (observed / ai / derived), sample-size badges, and definition tooltips on every chart build real trust. Most dashboards skip this.
  2. Mobile-first is enforced, not aspirational. The CLAUDE.md checklist, the dynamic yAxisWidth on BarChart, and the flex-col sm:flex-row header pattern all show evidence of iteration after real regressions.
  3. Progressive disclosure on the Plan page. Native <details> per day with format-chip previews is the right density tradeoff and respects touch.
  4. Adaptive min-N gating (Timing, Engagement, Strategy). minPostsForRange prevents the "one viral dawn post wins Early" trap — intellectually honest.
  5. Staleness banner + DataFooter. Telling the user when data is old, and where it came from, is the rare kind of transparency that earns trust over months.

→ Biggest opportunities

  1. Brand palette is under-deployed. The dashboard feels like shadcn, not Shikho. Indigo/pink/orange exist but only appear on login and the nav bar. Promote them into the primary chart color, the Weekly Verdict hero, focus rings, and KPI accents.
  2. Accessibility below WCAG AA. text-slate-400 on white (2.85:1) is used for actual content in ~80 places. Recharts axis ticks use the same colour. Focus rings missing from most interactive elements.
  3. Missing loading and error states. Cold loads look like a broken page for 300–1500ms. Add a skeleton per page type plus a friendly error.tsx.
  4. Eight pages is too many for one data source. Engagement / Timing / Strategy overlap in intent. Consolidate to five: Overview, Content, Timing (merged into Overview), Strategy+Plan (merged), Explore.
  5. No page template. Each page invents its own layout. Establish a canonical shape: Header → KPI strip (max 6) → 1 hero chart → 2-up secondary → detail. Everything feels the same shape thereafter.
03 — Detailed findings

Twelve things worth fixing, grouped by theme.

CriticalEff: S
WCAG AA failures for body-level content
text-slate-400 (#94a3b8) on white hits 2.85:1 — below the 4.5:1 AA bar. It's used for captions, sublabels, sample sizes, and axis ticks. Fix with a global find/replace to text-slate-500 (#64748b), which clears 4.6:1.
components/* · recharts axis fills
CriticalEff: S
No skeleton or error pages at app root
Cold loads render the page chrome and then blank panels for 300–1500ms. Add app/loading.tsx with a KPI-strip + chart skeleton, and an app/error.tsx that explains pipeline failure in plain English (not a stack trace).
app/loading.tsx · app/error.tsx
MajorEff: S
Brand palette isn't the primary chart color
Default bar color is cyan (#06b6d4); PALETTE leads with indigo then cyan. Re-order so shikho-indigo + shikho-pink are first and make shikho-indigo the default. One line in Donut.tsx + BarChart.tsx — visible lift everywhere.
components/BarChart.tsx:23 · Donut.tsx:4
MajorEff: M
Timing page uses two bar charts where a heatmap would work better
"When to post" is the canonical heatmap use case — day-of-week × hour, colour encoding mean reach. Two separate bar charts force users to cross-reference. A single 7×24 grid answers the question at a glance.
app/timing/page.tsx
MajorEff: S
6-slice donut in Engagement Breakdown
Cleveland & McGill: angle comparison breaks down past 3–4 slices. The 6-way reaction/comment/share breakdown should be a horizontal bar with descending sort — which the Engagement page already does correctly. The Overview "Engagement Mix" donut has the same problem and should also flip.
app/page.tsx:57
MajorEff: M
IA: 8 pages is ~3 too many
Engagement / Timing / Strategy all answer "what's working and why". Proposed: merge Trends into Overview (as a timeline strip), merge Plan into Strategy (as a "Next week" section), and keep Engagement + Timing + Reels as a unified "Content" section with three tabs.
components/Nav.tsx · overall IA
MajorEff: S
Overview KPI row has no visual hierarchy
All six KPIs are the same size, same colour, same weight. The single most important metric (Total Reach or Engagement Rate, depending on the week) should be hero-sized, with the other five as a secondary strip. See the Overview mockup below.
app/page.tsx
MinorEff: S
"Rendered {timestamp}" is a lie
The timestamp shown on every PageHeader is the render time, not the data fetch time. After 5 minutes of cache, the label still updates every reload even though the numbers don't. Rename to "Data as of" and wire to the Sheets fetch time from getRunStatus().
components/PageHeader.tsx
MinorEff: S
Avg Retention Curve on Reels is 60 bars
Retention is a continuous curve. 60 bars reads as 60 discrete events. Flip to TrendChart variant="line" with no dots — the curve shape is the insight.
app/reels/page.tsx:336
MinorEff: S
colorByIndex overused on value-comparison charts
Every bar a different colour signals categorical unrelatedness. On a ranked bar chart, it fights the sort. Reserve colorByIndex for legends where colour is cross-referenced elsewhere (Funnel, Spotlight type). Default value-comparison charts to a single brand colour.
BarChart.tsx · all consumers
PolishEff: S
Kind "text badge" was removed — keep going
Good: the left-border stripe now carries kind. But the three colors (cyan / indigo / violet) aren't explained anywhere the user sees. Add a tiny legend once in the DataFooter: "Chart edge colours — cyan: observed · indigo: AI-classified · violet: derived."
components/DataFooter.tsx
PolishEff: S
Typography is all-Inter-all-weights
One sans works, but big numbers deserve a voice. Introduce a display face (or just Inter's tighter tracking + font-variant-numeric:tabular-nums) for KPI values, Weekly Verdict headline, and page titles. Keeps body copy honest while making the metrics pop.
globals.css · tailwind.config.ts

05 — Visual system recommendations

From "Tailwind default" to "Shikho dashboard."

The brand palette and logo already exist. This section applies them as a consistent visual system, with tokens you can drop into tailwind.config.ts.

Lead with shikho-indigo as the primary chart colour. Use pink + orange as accent/alert. Reserve cyan/green/violet for multi-series legends only.

Page template

① Staleness banner (when warn/crit)
② Page header · title · data-as-of · range picker
③ Hero metric or verdict (one big thing)
④ Secondary KPI strip · max 5 cards

⑥ Two-up secondary · related context
⑦ Detail / table · for power users
Every page should answer this template. Reels, Trends, Explore each quietly violate it.

Loading & empty

Ship the skeleton as app/loading.tsx. Same shape as the page template. Content appears in place with zero layout shift.
