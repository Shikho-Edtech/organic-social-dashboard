05 · Decision log · for DECISIONS.md

Calls I made, and why

One paragraph each. Copy-paste directly.

Severity encoding uses border + icon tint, not a flooded background.
The current StalenessBanner uses bg-amber-50 and bg-rose-50 as full fills. Kept the soft fill but reduced contrast of the border so the banner sits calmly above PageHeader without competing with the Weekly Verdict hero beneath it. crit uses red-300/red-50 (not rose-) to stay within the Tailwind defaults you named in §4.
Banner is a <button>, not a <details>.
<details> would be cleaner semantically but gives up the aria-live="polite" announcement on state change (screen readers handle details as disclosure, not status). button + aria-expanded with the expansion rendered conditionally lets the banner keep role="status", which matters when the severity changes between renders. Entire row is the tap target (>44 px). Enter/Space toggle; Escape collapses.
ai-disabled is slate-with-indigo-accent, not a third severity color.
§5.4 calls out "neither red nor amber". Picked slate-300 border + slate-100 fill + indigo icon/link to encode "this is a product state, not a failure state." The indigo also ties visually to the empty-state ring so a reader can see the two surfaces are the same family at a glance. Explicitly avoided a teal/cyan "info" blue because that reads as a system message rather than an intentional off-switch.
Detail panel: stacked cards at 360, table at 1280.
At 360 px the table's four columns can't hold their shape even with overflow-x-auto — it either scrolls (per CLAUDE.md, bad for tabular nav) or the age column gets squeezed unreadable. Stacked rows keep stage name + status chip + last-success date + age visible in one glance without ever forcing horizontal scroll. The chip position and severity vocabulary is identical between the two shapes so the reader learns the system once.
Banner seam suppressed when expanded.
When the detail panel opens, the button's border-b becomes transparent and the panel's top border is slate-200 (not the severity color) so the two elements read as one continuous card. Prevents the "two boxes bolted together" look when the panel is open.
Empty state — two instances, one component.
Every visual affordance (card width, padding, indigo accent ring, pill position, button placement, "how to turn it back on" section) is byte-identical between /strategy and /plan. Only three strings change: the stage noun ("diagnosis" → "calendar"), the page read-source description, and the three env-var names. This is what "a reader should recognize the pattern instantly across both pages" translates to in code.
Card width capped at max-w-2xl even on 1280.
Widening a "disabled state" card to fill desktop would telegraph "this is the whole page now" — which feels like a failure state. Keeping it narrow and centered frames the empty state as an intentional inset, with the rest of the page header and nav clearly still working around it. Density isn't the point here; calm is.
Archival mode is a text line, not a banner.
You asked for "nothing fancy" and the spec specifically calls it a persistent slate-500 text-link banner. Rendered as a single-row line with only a bottom border — no box, no icon tint, no fill. It reads as a breadcrumb. The "Return to live view" action is a plain text button with underline on hover, matching DataFooter's treatment of secondary links.
Copy always leads with the stage by name.
"AI diagnosis is 9 days old" not "Strategy view may be out of date". The reader is the one person who can fix the pipeline — naming the stage makes the next action (check the stage's secrets in GH Actions) obvious. When two stages are stale simultaneously, the banner leads with the worst-affected by age, and the detail panel shows the full list.
No emoji, no alarm iconography, no animation.
Per §3 explicitly. The "empty state" pill uses a 1.5 px solid dot rather than a spinning icon; the archival mode uses a static rotate-counterclockwise glyph at slate-400; the banner chevron is a static rotate transform, not an animation. Everything respects the "data-honest, quiet, confident" mood.
