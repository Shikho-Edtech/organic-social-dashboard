Why this exists

Voice transcript 2026-05-01: "the engagement page has too many charts that say similar things. I want fewer, more decision-driving views." Current page is 959 lines, ~12 charts, 3 of which are KPI strips. R2 target: 5 anchored views, each answering one specific question.

Proposed

5 anchored decision-driving views

2 · Format × Hour Heatmap — kept (already best-in-class)
3 · Funnel Engagement — kept (mechanical math, doesn't fit elsewhere)

5 · Comparison Lab — optional drill-down: pick any 2 dimensions for cross-tab

~600 lines saved. One axis switcher replaces 8 stacked charts.

View 1 — Performance by Dimension Consolidates 8 charts → 1

The core consolidation

Eight bar charts (4 dimensions × 2 metrics) become one chart with two switchers: a dimension pill row + a metric pill row. The active combination renders below.

Dimension switcher:
Pillar Format Hook Caption Tone Spotlight Type
Metric switcher:
Reach Engagement Rate Shares Quality Engagement
[bar chart — Reach × Pillar — same shape as today's BarChartBase]
Decision: use page-level ?metric URL param as default for the metric switcher. This already exists site-wide. New URL param: ?eng_dim for the dimension. Both mirror Reels' tab pattern.
Crown badge: the "Best X" KPI strip moves inline with this card — when you flip to Pillar, the KPI line above the chart shows "Live Class / Exam Prep — 412k avg reach (best)". The whole concept of a separate "Best" strip dies.

What we're NOT changing (guardrails)

  • Engagement page is mechanical math — no AI credits. Reaffirmed; all 5 views above are pure aggregation, no LLM call.
  • Existing 4-metric URL param contract (?metric=reach etc.) preserved as the metric switcher's source of truth.
  • Format × Hour heatmap kept verbatim.
  • Winning Pattern data shape from the current page reused — just rendered differently.

Migration path (proposed)

  1. This wireframe — review with user, capture feedback, iterate to v2 if needed.
  2. QA env trial — wire up View 1 (the big consolidation) on a feature-flagged route (e.g. /engagement?layout=r2). Real data, real interactions, but old layout still default.
  3. Operator A/B — user tries both for ~1 week. If R2 wins on time-to-insight, promote to default.
  4. Cutover — old 8-chart stack deleted in one commit; URL params preserved.

Open questions for v2

  • Does the dimension switcher need to remember last-active per session, or always default to Pillar? (Today's ?metric persists via URL — same here for ?eng_dim?)
  • Comparison Lab default — collapsed or expanded? Collapsed = cleaner; expanded = discoverable but adds visual noise.
  • Winning Pattern Cards — 3-up always, or responsive 1-up on mobile?
  • Do we keep "isLowConfidence" tags on the bars when sample size is below the floor? (Today the chart hides them entirely; Comparison Lab might show them dimmed instead.)
